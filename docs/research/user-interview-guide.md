# 用户访谈指南 — 30 户验证

> 目标：在写一行业务代码前，验证 4 个关键假设；产出可执行的产品决策。
> 节奏：2 周内完成 30 户访谈（个人博主 15 + 轻 MCN 15）。
> 输出：每周 1 份汇总报告，2 周后立项 / 调整 / Pivot 三选一。

---

## 1. 待验证的 4 个关键假设

| # | 假设 | 证伪即调整 |
|---|---|---|
| H1 | 博主和 MCN 编辑愿意在**手机端**做选题、写作、审稿、看数据 | 改 Web 优先，小程序仅做填报与提醒 |
| H2 | 跳 XHS App 的"半自动发布"体感可接受（保存+复制+跳转 ≈ 一键） | 退回纯协作工具，去掉"一键发布"承诺 |
| H3 | 个人 29 元/月、团队 99 元/席 的价格带能成立 | 重做定价或换商业模式（按次收费/按内容效果分成） |
| H4 | 现有工具的核心痛点不是功能不够，而是**重 + 贵 + 不协作** | 砍掉重 SaaS 路线，押注移动协作 |

每户访谈结束都要打分 4 个假设各自的支持度（-2~+2），最后汇总。

## 2. 招募

### 2.1 数量与画像

| 类别 | 数量 | 画像要求 |
|---|---|---|
| 起号期个人博主 | 5 | 粉丝 <5k，1~2 个号，发布 <30 篇 |
| 成长期个人博主 | 5 | 粉丝 5k~50k，2~3 个号，半年内变现过 |
| 头部个人博主 | 5 | 粉丝 >50k，有助理或 1 人小团队 |
| 自营品牌矩阵 | 5 | 自营商家用 3~10 号矩阵导流 |
| MCN/代运营机构 | 5 | 管 10~50 个达人号 |
| 投放/广告团队 | 5 | 媒介岗，做小红书投放与选号 |

### 2.2 招募渠道

- 小红书：发"约访谈 + 50 元红包"笔记（标签 #运营 #博主成长），优先评论区私聊。
- 微信群：垂类博主群、MCN 内部群（找朋友拉群）。
- 即刻 / V2EX：发"做小红书工具，招访谈"。
- 老用户冷启：竞品（千瓜/矩阵通）社群里看活跃用户。
- LinkedIn / 脉脉：MCN 运营总监级。

### 2.3 激励

- 个人 50 元 / 头部 100 元 / MCN 决策人 200 元。
- 内测期免费 6 个月 Pro。
- 一对一深访不超过 45 分钟，30 分钟版本红包减半。

## 3. 访谈结构（45 分钟）

### 3.1 第 1 段 · 暖场与背景（5 分钟）

- 自我介绍：我们在做一个面向博主和运营的协作工具，今天不卖东西，只想听你怎么干活的。
- 录音授权（口头确认；不录视频；仅内部使用；可随时叫停）。
- 基本信息：你是个人还是团队、几个号、主营赛道、起号多久。

> ⚠️ 不出现"小程序""矩阵管理""AI 写作"等产品词；让用户先描述自己。

### 3.2 第 2 段 · 任务流（15 分钟）

请用户**讲他上周做的最近一篇笔记的全流程**（行为访谈，不是态度访谈）：

1. 选题怎么来？看哪些工具？看什么样的爆文？
2. 文案怎么写？用 AI 吗？哪个？怎么用？
3. 图片/视频在哪儿处理？
4. 发之前会检查什么？违禁词怎么发现？
5. 发布以后怎么跟数据？多久回看一次？
6. 私信怎么处理？怎么转化？
7. **最近一次让你想砸手机的环节是什么？**（关键问题）

每个环节让 ta 估一下用时 + 频率 + 工具名。落到表里：

| 环节 | 工具 | 用时 | 频率 | 不满 |
|---|---|---|---|---|
| 选题 | … | … | … | … |
| … | | | | |

### 3.3 第 3 段 · 当前工具（10 分钟）

- 列出你正在用的工具（含 Excel / 微信群 / 备忘录）。
- 每个工具：你最喜欢什么？最恨什么？为它付费了吗？多少？
- **如果明天它没了，你怎么办？**（粘性测试）
- 你在手机上的工具占比 vs 电脑？为什么？

### 3.4 第 4 段 · 半自动发布演示（5 分钟，仅对承认有此痛点的人）

口头描述（不演示）：

> "假如有这么个小程序：你在里面写好图文，点一个按钮，图片自动存到相册、文案自动复制、小红书 App 自动打开发布页，你只需要粘贴 + 选图 + 点发布。"

问：
1. 这跟你现在的发布有差异吗？你愿意试吗？
2. 哪一步是你最在意的省时点？
3. 如果跳转有时候唤起失败，你能接受吗？

### 3.5 第 5 段 · 价格意愿（5 分钟）

引导用户**主动报价**而不是给选项：

- "你愿意为今天聊的最痛的那个环节付多少？"
- 然后给参考锚点："比如千瓜要几千一年，竞品矩阵通几千一月，你怎么看？"
- 拆出来：单点工具（写作 / 看数据 / 审稿）愿付价 vs 套件价。

### 3.6 第 6 段 · 收尾（5 分钟）

- 你最希望它做什么 / 千万不要做什么？
- 我们做出来后愿意第一批试用并反馈吗？
- 可以推荐 2~3 个同行给我们访谈吗？（关键 — 滚雪球）

## 4. 给访谈员的话术清单

**鼓励具体而不是抽象**
- ❌ "你觉得 AI 写作有用吗？" 
- ✅ "你上次用 AI 写文案是什么时候？给我看下当时的对话？"

**捕捉真痛点**
- "上周哪天你心情最差？工作上发生了什么？"
- "你为这件事熬夜过吗？"

**避开诱导**
- ❌ "如果有个工具能 X，你会用吗？"（人人都说会）
- ✅ "你现在怎么解决 X？花多少时间？"

**价格不要给选择题**
- ❌ "29、99、299，你选哪个？"
- ✅ "你愿意付多少？" → 沉默 → 让用户先开口。

**留白与追问**
- 用户说完后停 3 秒；不要急着提下一个问题；最有用的信息往往在第二段话里。

## 6. 汇总与决策

### 6.1 每周报告模板

- 完成访谈数 / 招募漏斗
- 4 个假设的当前打分均值 + 分布
- Top 3 真痛点（合并同类项后的频次）
- 价格意愿分布直方图
- 关键金句 5 条
- 下周招募计划

### 6.2 假设结论判定

| 平均分 | 解读 | 行动 |
|---|---|---|
| ≥ +1.0 | 强支持 | 按 P0 路线推进 |
| 0 ~ +1.0 | 弱支持 | 需要在 MVP 里设计 A/B 验证 |
| -1.0 ~ 0 | 弱反对 | 调整产品形态再做一轮 5 户验证 |
| ≤ -1.0 | 反对 | 该模块砍掉或重设计 |

### 6.3 立项门槛

**通过条件（4 条全满足才推进 P0 开发）**：
1. H1 H2 H3 H4 中至少 3 个 ≥ +0.5
2. 30 户中至少 12 户愿意做"第一批内测"
3. 至少 5 户报出的"套件愿付价" ≥ 49 元/月
4. Top 3 痛点的合并频次 ≥ 60%（说明痛点收敛）

## 8. 风险与对策

| 风险 | 对策 |
|---|---|
| 招募来的都是友谊用户、不说真话 | 渠道分散 + 红包 + 行为访谈优先态度访谈 |
| 用户描述与实际行为不一致 | 让用户当场打开手机演示，能录屏更好 |
| 价格问题撒谎（说会买实际不买） | 同时问"上次为同类工具实际花了多少钱" |
| 30 户样本不够代表性 | 第二轮再补 20 户量化问卷验证关键结论 |
