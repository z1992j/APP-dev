## 1. 产品概述

### 1.1 一句话定义

**面向小红书博主与轻量 MCN 的"移动协作 + AI 写作 + 半自动发布"工具型微信小程序。**

### 1.2 价值主张

| 用户场景 | 现状 | 我们的方案 |
|---|---|---|
| 起号博主一个人扛 3 个号 | Excel + 微信群 + 切登录，AI 写作工具碎片化 | 一个小程序写作、检查、排期、看数据 |
| MCN 编辑 / 运营负责 10+ 账号 | 矩阵通几千一月，PC 端用不顺手 | 月费 ¥99/席，移动协作，编辑/审稿/任务三角 |
| 品牌方矩阵号导流 | 文案靠人肉、违禁词被限流 | AI 写作 + 三层违禁词检测，发布前可见 |

### 1.3 核心差异化

1. **移动端协作优先** — 行业内 PC SaaS 一统天下，移动空白带。
2. **合规发布** — 不做"代登录代发布"，半自动闭环（保存图 + 复制文案 + 跳 XHS），绕开小红书风控与微信审核红线。
3. **AI 资产化** — 按垂类做 prompt 资产，账号档案影响 AI 输出，越用越懂博主。
4. **价格友好** — 个人 ¥29/月、团队 ¥99/席/月，比矩阵通/千瓜便宜一个数量级。

### 1.4 不做的事（红线清单）

- ❌ 自动登录小红书账号
- ❌ 代用户自动发布、自动回复私信
- ❌ 云手机托管 / Cookie 池 / 指纹浏览器
- ❌ 大规模爬取小红书公开数据
- ❌ web-view 嵌入小红书站点
- ❌ 在文案 / 推广中出现"自动发布""绕过风控""一键代运营"等词

## 3. 用户画像

- 付费意愿：¥29~49 / 月

- 矩阵 25 个账号，月毛利 ¥15w
- 痛点：每个达人发什么不知道、爆款不会复用、矩阵通用着重也贵、Excel 满天飞

## 4. 核心用户旅程

### 4.1 起号博主的一天

```
早晨通勤
  ↓
  打开小程序，看「今日推荐」灵感卡片
  ↓
  收藏 2 条到选题池
  ↓
午休
  ↓
  从选题池选一条，进 AI 写作工作台
  ↓
  填主题 / 选账号 / 选风格 → 生成
  ↓
  挑标题、用"改写"工具调整正文
  ↓
  跳转草稿编辑器，上传 8 张图
  ↓
晚上
  ↓
  点"检查"，违禁词面板提示 2 处替换
  ↓
  点"发布到小红书"
  ↓
  跳转 H5 中转页 → 自动唤起小红书 App
  ↓
  在小红书内粘贴文案 + 选图 + 发布
  ↓
  回到小程序，粘贴笔记链接回填
  ↓
睡前
  ↓
  「今日填报」推送弹窗 → 30 秒填完
  ↓
  看一眼数据看板
```

### 4.2 关键漏斗

```
注册 → 创建账号档案 → 写第一篇 → 检查通过 → 发起 handoff → 回填发布 → 填报数据
 100%     70%             45%         40%          35%             25%           20%
```

任一环节断崖 > 50% 都触发产品复盘。

## 5. 功能需求

> 每条 FR 都包含：描述、用户故事、验收标准、依赖、风险。
> 详细字段、SQL、API 见 [P0 详设](../design/p0-detailed-design.md)；本节只列需求。

### 5.1 FR-1 · 用户与账号档案

**描述** 微信登录 + 手机号绑定 + 多账号档案管理。

**用户故事**
- 作为博主，我想用微信一键登录，不再注册账号密码
- 作为博主，我想给每个小红书号建一个"档案"，记录人设
- 作为博主，我想在切换不同账号上下文时，AI 写作也跟着变风格

**功能需求**
- FR-1.1 微信登录 / 手机号一键登录（兜底）
- FR-1.2 账号档案 CRUD：昵称、头像、主页链接、赛道、人设（性别、年龄、城市、自我介绍、口头禅、禁词）
- FR-1.3 主账号 / 子账号档案区分（v1.0 全部平权，P1 加角色）
- FR-1.4 档案数量限制：免费 1，个人 Pro 5，团队 Starter 10，团队 Pro 50

**验收标准**
- ✅ 新用户冷启动 → 微信授权 → 创建第一个档案 → ≤90s
- ✅ 切换档案后，AI 写作的 system prompt 立即更新
- ✅ 删除档案不影响历史草稿（关联保留 archived 状态）

**依赖** 微信开放平台、企业小程序主体
**风险** 个人主体无法开通虚拟支付 → 一期强制企业主体

---

### 5.2 FR-2 · 灵感选题

**描述** 关键词搜索 + 推荐 + 选题池。

**用户故事**
- 作为博主，我想根据关键词找近期爆文做参考
- 作为运营，我想把找到的好选题收藏到池子里

**功能需求**
- FR-2.1 搜索：关键词 + 赛道筛选 + 时间窗口（7/30/90 天）
- FR-2.2 推荐：基于账号档案 vertical 个性化首页
- FR-2.3 笔记详情卡片：封面、标题、摘要（≤200 字）、互动数、博主、tag
- FR-2.4 选题池：收藏、分组、备注
- FR-2.5 兜底：粘贴笔记链接 → 服务端 oembed 解析
- FR-2.6 频次限制：免费 10 次/天，Pro 100 次/天

**验收标准**
- ✅ 搜索响应 P95 < 1.5s（数据源 + 缓存）
- ✅ 数据源失败时显示降级文案 + LLM 兜底选题清单
- ✅ 卡片可"加入选题池""用这条写作"双 CTA
- ✅ 笔记摘要不存储全文（版权合规）

**依赖** 数据源 BD（千瓜 / 新红 至少 1 家）
**风险** 数据源断供 → 必须签 2 家备份；自有用户填报作为长期主路径

---

### 5.3 FR-3 · AI 写作工作台

**描述** 多账号 fan-out + 流式生成 + 局部改写。

**用户故事**
- 作为博主，我想根据主题快速生成 5 个标题候选 + 一篇正文
- 作为运营，我想给 5 个矩阵号一稿生成 5 个不同人设版本
- 作为博主，我想选中文本一键改写、扩写、加 emoji

**功能需求**
- FR-3.1 输入：主题、目标账号（1~N）、风格（种草/干货/吐槽/故事）、字数档（80/200/500/1000）、参考笔记（可选）
- FR-3.2 输出：5 个标题候选 + 1 篇正文 + 推荐 hashtag
- FR-3.3 流式返回（SSE）：标题逐个出现，正文逐字流出
- FR-3.4 多账号 fan-out：选 N 个账号 → 生成 N 个版本 → 写入 N 个独立草稿
- FR-3.5 局部改写：选中文本 → 改写 / 扩写 / 缩写 / 换风格 / 加 emoji
- FR-3.6 prompt caching：vertical 知识 + persona + 风格三块走 cache
- FR-3.7 多供应商容错：Claude 4.6 主，Qwen-Max 备
- FR-3.8 配额：免费 3 次/天，个人 Pro 100 次/天，团队 1000 次/天/席；超额按 token 计费

**验收标准**
- ✅ TTFB ≤ 1s
- ✅ 完整 200 字正文 ≤ 8s 出完
- ✅ AI 失败重试 2 次后切备供应商
- ✅ 单次写作平均成本 ≤ ¥0.12（首篇）/ ≤ ¥0.08（缓存命中）
- ✅ 多账号 fan-out 5 个版本 ≤ 15s
- ✅ 输出符合小红书风格（标题 ≤20 字、emoji 装饰、3~6 hashtag）

**依赖** Claude API、Qwen API、prompt caching 能力
**风险** AI 成本超支 → 配额 + caching + 模型路由 + 用量计费三件套

---

### 5.4 FR-4 · 违禁词与内容安全

**描述** 三层检测：自有词库 + 微信安全 API + LLM 上下文判断。

**用户故事**
- 作为博主，我想在发布前看到所有违禁词风险，并能一键替换

**功能需求**
- FR-4.1 L1 自有词库：极限词、医疗、金融、低俗、平台风险 ≥ 1500 条
- FR-4.2 L2 微信内容安全 API（msgSecCheck + imgSecCheck）
- FR-4.3 L3 LLM 上下文判断（医疗暗示、广告法擦边、敏感品类）
- FR-4.4 内联标记：编辑器内文本下划线 + 颜色（红/黄/灰）
- FR-4.5 改写建议：每个命中词给建议词 + 一键替换
- FR-4.6 全部替换 / 暂时忽略
- FR-4.7 词库热更新 + 客户端缓存（版本号 + etag）

**验收标准**
- ✅ L1 检测响应 P95 < 50ms（Trie 索引）
- ✅ L2 拦截命中触发 modal + 改写建议（不允许强发）
- ✅ L3 在 L1/L2 通过后异步执行，不阻塞编辑
- ✅ 词库可在管理后台 30 分钟内全量生效

**依赖** 微信内容安全 API、LLM API
**风险** L1 误判率高 → 提供"忽略"开关 + 后台埋点收集误判样本迭代

### 5.5 FR-5 · 草稿与媒体管理

**描述** 图文/视频草稿 CRUD、媒体上传、状态流转。

**用户故事**
- 作为博主，我想保存多个草稿，按状态过滤
- 作为博主，我想上传 18 张图 + 一段视频，自动压缩

**功能需求**
- FR-5.1 草稿 CRUD：标题、正文、媒体、hashtag、目标账号
- FR-5.2 图片上传：单张 ≤10MB，客户端压缩到 webp，直传 COS，最多 18 张
- FR-5.3 视频上传：≤500MB，分片上传，客户端压缩
- FR-5.4 单图编辑：裁剪、加水印、贴纸
- FR-5.5 自动保存：每 5s 一次，离线缓存
- FR-5.6 状态机：draft → scheduled → handed_off → published → archived
- FR-5.7 列表视图 + 日历视图 + 状态筛选

**验收标准**
- ✅ 图片上传 P95 ≤ 3s（4G 网络，单张 < 2MB）
- ✅ 500MB 视频上传可断点续传
- ✅ 自动保存丢失数据 = 0（哪怕 app 杀进程）
- ✅ 状态变更全部落 audit_log

**依赖** 腾讯云 COS、ffmpeg 服务
**风险** wx.uploadFile 10MB 限制 → 走 COS 直传 SDK 绕开

### 5.6 FR-6 · 排期与提醒

**描述** 设定发布时间 + 订阅消息提醒。

**用户故事**
- 作为博主，我想把草稿安排在明天晚 8 点，到点提醒我去发

**功能需求**
- FR-6.1 草稿设定 schedule_at（任意未来时间）
- FR-6.2 日历视图：月/周
- FR-6.3 订阅消息：到时间推"该发布啦"
- FR-6.4 一次性授权额度管理（subscribe_token 表）：用户首次创建排期时引导"批量授权 5 次"
- FR-6.5 提醒可重复申请（消费完一次提示重订）

**验收标准**
- ✅ 推送送达率 ≥ 95%
- ✅ 跨时区行为正确（统一存 UTC，客户端按 local 显示）
- ✅ 日历支持手势拖拽改时间

**依赖** 微信订阅消息模板审核通过
**风险** 一次授权一次推送的限制 → UX 引导 + 重订模板

### 5.7 FR-7 · 跳转小红书（半自动发布）

**描述** 保存图片 + 复制文案 + 唤起小红书 App。

**用户故事**
- 作为博主，我想点一个按钮，把图片和文案准备好，直接跳到小红书发布页

**功能需求**
- FR-7.1 点"发布到小红书" → 弹确认 → 执行三步：
  1. wx.saveImageToPhotosAlbum 批量保存图片
  2. wx.setClipboardData 写入"标题\n正文\n#tag"
  3. wx.openLink 打开自家 H5 中转页
- FR-7.2 H5 中转页（业务域名）：Universal Link / intent:// 自动唤起小红书
- FR-7.3 唤起失败降级：显示步骤指引 + 重试按钮 + 教程视频
- FR-7.4 草稿标记为 handed_off + 落 audit
- FR-7.5 兼容性矩阵：iOS / Android × XHS 三个主版本必须 100% 跑通"保存+复制+跳转"，唤起成功率 ≥ 70%

**验收标准**
- ✅ iOS + XHS 8.30+ 唤起成功率 ≥ 80%
- ✅ Android + XHS 8.30+ 唤起成功率 ≥ 75%
- ✅ 唤起失败时降级页加载 ≤ 1.5s
- ✅ 用户能在 30s 内完成"在 XHS 内粘贴 + 选图 + 发布"

**依赖** 业务域名 + ICP + apple-app-site-association 配置
**风险** XHS 改 scheme 或 Universal Link → 埋点监控成功率 + 兼容性矩阵自动告警

### 5.8 FR-8 · 发布回填

**描述** 用户回填发布的 XHS 笔记链接 → 服务端解析 metrics。

**用户故事**
- 作为博主，发完笔记后我想回到小程序，记录这条笔记的链接，方便后续看数据

**功能需求**
- FR-8.1 草稿列表"待回填"卡片提示
- FR-8.2 输入 XHS 笔记链接 → 服务端 oembed 解析（封面、标题、首批互动数）
- FR-8.3 草稿状态 → published；published_url + 初始 data_point 写入

**验收标准**
- ✅ 解析失败可手动跳过（仅记录链接）
- ✅ 链接合法性校验（host = xiaohongshu.com / xhslink.com）

**依赖** 后端解析 worker
**风险** XHS 反爬升级 → 退化为"仅存链接，metrics 靠用户填报"

### 5.9 FR-9 · 数据看板

**描述** 单账号 + 全矩阵看板 + 每日填报。

**用户故事**
- 作为博主，我想每天 30 秒填一次数据，看趋势
- 作为运营，我想看全矩阵汇总

**功能需求**
- FR-9.1 每日填报：21:00 订阅消息推送 + 进入小程序首屏弹窗（每天一次）
- FR-9.2 填报字段：粉丝总数、当日曝光、点赞、收藏、评论、私信、发布数
- FR-9.3 智能预填：用上次值 + 自动差分
- FR-9.4 单账号视图：折线图（粉丝 / 曝光 / 互动率）+ 笔记列表 + 爆款识别
- FR-9.5 全矩阵汇总：网格 + 总和 + Top 5
- FR-9.6 时间窗口：7/30/90 天切换
- FR-9.7 周报 / 月报：PDF/PNG 导出

**验收标准**
- ✅ 填报弹窗 30s 完成率 ≥ 60%
- ✅ 7 日趋势查询 P95 ≤ 500ms
- ✅ 报表导出 ≤ 5s

**依赖** ECharts for 小程序、报表服务
**风险** 用户填报数据质量 → 智能预填 + 离群点提示

### 5.10 FR-10 · 订阅与计费

**描述** 套餐选择 + 微信支付 + 配额管理。

**用户故事**
- 作为博主，AI 配额用完时我想能升级到 Pro
- 作为团队 owner，我想买 5 席并邀请成员

**功能需求**

## 7. 非功能性需求

| 类别 | 要求 |
|---|---|
| **性能** | 冷启动 < 1.5s；非 AI 接口 P95 < 500ms；AI TTFB < 1s |
| **可用性** | 月度 SLA 99.9%；核心接口降级路径完备 |
| **安全** | 全站 HTTPS；JWT + Refresh；接口 HMAC；KMS 管密钥；不存任何 XHS cookie/账号密码 |
| **合规** | ICP 备案；UGC msgSecCheck/imgSecCheck；个保法（最小必要 / 可删除 / 可导出）；第三方 SDK 清单；隐私协议明示 |
| **兼容** | 微信版本 ≥ 8.0.27；iOS ≥ 14；Android ≥ 8 |
| **可观测** | 全链路 trace；前端 Sentry；后端 Prometheus + Grafana；业务漏斗埋点 |
| **数据** | 草稿永久保留；inspire_note 缓存 30 天；audit_log 1 年；data_point 永久 |
| **备份** | PG 每日全量 + WAL 持续；COS 跨区复制 |
